    2. Create `src/features/graph/client.ts` — the core Graph client module. This module exports two functions:
       - `graphFetch<T>(path, getToken, options?)` — single-page fetch with token injection and 429 retry
       - `graphFetchAll<T>(path, getToken, options?)` — full pagination loop that follows all `@odata.nextLink` pages

       Implementation details:

       ```typescript
       import type { GraphPagedResponse, GraphError } from "./types";

       const GRAPH_BASE = "https://graph.microsoft.com/v1.0";
       const MAX_RETRIES = 3;

       type GetTokenFn = () => Promise<string>;

       interface FetchOptions {
         method?: "GET" | "POST" | "PATCH" | "DELETE";
         body?: unknown;
         headers?: Record<string, string>;
       }
       ```

       `graphFetch` implementation:
       - Accept `path` (e.g., `/me/todo/lists`), `getToken` callback, and optional `FetchOptions`
       - Call `getToken()` to get a fresh Bearer token before each request
       - If `path` starts with `https://`, use it as the full URL (for nextLink/deltaLink URLs). Otherwise prepend `GRAPH_BASE`.
       - Set headers: `Authorization: Bearer ${token}`, `Content-Type: application/json`
       - If `body` is provided, JSON.stringify it
       - On response, check status:
         - `429`: Read `Retry-After` header (default to 5 seconds if missing). Wait that many seconds. Retry up to `MAX_RETRIES` times. If all retries exhausted, throw an error with the retry-after value.
         - `410`: Throw a specific error with code `"GoneError"` so callers (delta sync in Phase 2) can detect it
         - `401`: Throw an error (token was invalid — caller should re-auth)
         - `2xx`: Parse and return JSON as `T`
         - Other 4xx/5xx: Parse error body as `GraphError`, throw with `error.code` and `error.message`

       `graphFetchAll` implementation:
       - Call `graphFetch<GraphPagedResponse<T>>` for the initial path
       - Accumulate all `value` items into a single array
       - While the response has `@odata.nextLink`, call `graphFetch` with the full nextLink URL (it's an absolute URL — pass it directly as the path since it starts with `https://`)
       - Return `{ value: allItems, deltaLink: lastResponse["@odata.deltaLink"] }` — the deltaLink from the final page (if present) for delta sync use in Phase 2

       Return type for `graphFetchAll`:
       ```typescript
       interface GraphFetchAllResult<T> {
         value: T[];
         deltaLink?: string;
       }
       ```
